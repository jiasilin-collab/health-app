# 📱 Web应用测试指南

## 🚀 快速开始

### 方式1：访问本地Web应用（推荐）

**Mock API服务器已启动：**
- API地址：http://localhost:3000
- 健康检查：http://localhost:3000/health

**前端应用访问：**
```
http://localhost:3000
```

### 方式2：在手机浏览器中访问

#### 步骤1：获取电脑IP地址

**Windows：**
```bash
ipconfig
# 查找 IPv4 地址（例如：192.168.1.100）
```

**Mac/Linux：**
```bash
ifconfig
# 或
ip addr show
```

#### 步骤2：确保手机和电脑在同一网络

- 连接到同一WiFi
- 或通过手机热点连接

#### 步骤3：在手机浏览器访问

```
http://[你的电脑IP]:3000
```

**例如：**
```
http://192.168.1.100:3000
```

## 🔧 调试工具

### 浏览器开发者工具

**PC浏览器：**
1. 按F12打开开发者工具
2. 查看Console中的错误
3. 查看Network中的API请求

**手机浏览器（Chrome）：**
1. 启用USB调试
2. Chrome访问 `chrome://inspect`
3. 选择要调试的页面

### Mock API测试

使用curl测试API：

```bash
# 健康检查
curl http://localhost:3000/health

# 获取微量元素列表

# 获取食谱列表
curl http://localhost:3000/api/recipes

# 获取体质类型
curl http://localhost:3000/api/tcm/constitutions
```

## ⚠️ 常见问题

### Q1: 无法访问localhost:3000？

**解决方案：**
1. 确认Mock API服务器正在运行
2. 检查端口3000是否被占用
3. 尝试使用http://127.0.0.1:3000

### Q2: 手机无法访问电脑IP？

**解决方案：**
1. 确认手机和电脑在同一WiFi
2. 关闭防火墙或允许端口3000
3. 检查Windows防火墙设置

### Q3: 页面显示空白？

**解决方案：**
1. 打开浏览器开发者工具
2. 查看Console中的错误
3. 检查Network中的请求是否成功
4. 刷新页面

### Q4: 功能不正常？

**解决方案：**
1. 检查API是否返回正确数据
2. 检查前端代码是否正确处理响应
3. 查看Console中的JavaScript错误
4. 尝试清除缓存
